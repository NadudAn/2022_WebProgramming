관심 기술 상세 내용

먼저 웹 개발자와 백엔드의 정의에 대하여 알아보고, 백엔드 개발자와 자바 개발자가 알아야할 것들에 대해 알아보겠다.

  • 웹 개발자
    • HTTP 프로토콜을 커뮤니케이션 매체로 사용하는 웹 페이지, 웹 사이트 등 WWW기반 소프트웨어 개발자 또는 엔지니어.
    • 웹 디자인, 정보설계, 사용자 인터페이스 설계, 프로젝트 관리, 웹 서버 및 데이터베이스 관리, 웹 페이지 코딩 및 프로그래밍 관련 기술을 지님.
    • 전문성을 기준으로 프론트엔드와 백엔드로 분류.

  • 백엔드(Back-end)
    • 사용자에게 보이지 않는 비즈니스 로직 프로그래밍을 하는 개발자.
    • UI나 GUI로 구성된 화면의 통신/요청에 대해 DB 또는 Interface 등을 통해 시스템 구성 실체에 접근.
    • 서버, 응용 프로그램, 데이터베이스를 작동시키는 업무 수행
    • 파이썬, Node.js, PHP7, Ruby, C#, Java, Go 등의 언어 주로 사용.

    • 세부 직무
      • 서버 관리자: 서버 환경설정, 서버구성 관리
      • DB 관리자: DB 설계, Data 관리 및 최적화
      • Back-end Developer: 서버 자체를 코딩
백엔드 개발자가 알아야할 것들
  • API Development
    • Rest API는 오랫동안 API의 표준처럼 사용되어 왔고 서버 개발의 전반적인 흐름을 이해하는데 도움 됨.
  • Languange & Web Framework
    • NodeJS (JavaScript, TypeScript), Ruby On Rails, Python Django, Spring Boot(Java, Kotlin), golang 등
  • Server
    • AWS의 EC2 Instance에 올라가는 이미지들: 주로 Linux 기반의 OS.
    • AWS Elastic Beanstalk: 기본 인프라 구성, 배포, 애플리케이션 버전 제어, 자동 확장 기능까지 전체 시스템을 구축.
  • Network
    • 서버 동작 전체를 잘 이해 하기 위해서 네트워크에 대한 기본 지식 필요.
    • HTTP, DNS, EC2에 접속하기 위한 SSH 연결, 서버가 올라가기 위한 기본 네트워크 구성 (VPC, Routing Table, Subnet)등
  • DB
  • Distributed Storage
    • 서비스에 필요한 서비스를 저장하는 공간.
  • Basic Frontend Skill
자바 개발자가 알아야할 것들
  • Learn Spring Framework(Spring Boot)
    • 요즘에는 자바 개발자들이라면 피할 수 없는 프레임워크가 바로 스프링 프레임워크. 대부분의 회사들이 스프링 프레임워크를 이용하여 개발하는 것을 선호. 스프링에는 Spring MVC, Spring Boot, Spring Cloud 등의 프로젝트가 있는데, 이를 통해 웹 어플리케이션, REST API, microservice를 개발하고 있음.
    • 또한 의존주입과 같은 모범사례를 장려하고 응용 소프트웨어의 테스트 가능성을 높임. 이는 최신 소프트웨어의 핵심 요구사항.
  • Learn Unit Testing
    • 좋은 자바 개발자와 그저그런 평범한 자바 개발자를 구별하는 한 가지는 바로 단위 테스트 기술.
    • 전문적인 자바 개발자는 대부분 항상 작성한 코드의 테스트를 위해 단위테스트 코드를 작성. 이들의 코드를 보게 된다면 대부분은 응용 프로그램 작성 코드와 그에 맞는 단위테스트 코드를 볼 수 있을 것.
    • 자바개발자가 사용할 수 있는 단위테스트, 통합테스트 및 자동화 테스트를 위한 여러가지 툴들과 함께 테스트도 오래오래 발전해옴.
    • 추천 라이브러리: JUnit
  • Learn APIs and Libraries
    • 자바는 전세계에서 가장 인기 있고 오래된 프로그래밍 언어. 아주 다양한 라이브러리와 api가 존재. 당연히, 그 모든 것을 공부하기란 어렵고 쉽지 않을 것. 하지만 몇몇 중요 API에 대한 경험이나 존재 자체에 대해서는 인지하고 있어야 함.
    • JSON processing Jackson, Gson / unit testing like Mockito, JUnit
  • Learn the JVM`s Internals
    • JVM의 다른 부분은 무엇인지, 어떻게 동작되는지, JIT 컴파일러, JVM 옵션들, Garbage Collection 과 같은 컬렉터들.
    • JVM에 대해 잘 안다면, rockstar 자바 개발자와 같이 건장하고 성능좋은 자바 응용프로그램을 만들 수 있을 것, 성능에 장애가 있는 부분을 빠르게 파악할 수 있는 능력이 생길 것.
    • 추천 책: The Definitive Guide to Java Performance
  • Learn Design Patterns
    • 디자인 패턴을 알고 이를 코드에 적용시킴으로써 당신이 개발한 응용프로그램의 유연함이 향후 코드를 수정하는 데 있어서 개선될 가능성이 높아짐.
    • 디자인 패턴을 도입한다면 다른 개발자들이 해당 코드를 수정할 때 디자인 패턴에 익숙하다면 다른 개발자가 작성한 코드라고 할지라도 더 효과적으로 이해할 수 있으며 이는 전체 코드 퀄리티의 향상을 도모.
    • 그러나 너무 코드 작성에만 기울이지 말고 로직을 이해하고 창의적으로 해결해야 함.
  • Learn DevOps Tools
    • 모던 자바 개발자는 최소한 CI / CD(Continuous Integration / Continuous Deployment) 에 대해서는 알고 있어야 하며 Jenkins가 어떻게 이를 지원할 수 있는지 알아야 함.
    • 시니어 자바 개발자들에게도 이는 더욱 중요. 가이드라인을 정하고 프로젝트 코드를 셋팅하고 환경을 설정하고 빌드스크립트를 짜는 중책을 맡았기 때문.
    • 데봅스에 관한 일반적인 내용과 Docker, Chef, Kubernetes 와 같은 도구와 Maven 및 Jenkins와 관련하여 알아보기.
    • 온라인 강의 추천 사이트: Udemy
  • Learn Kotlin
    • 코틀린은 InteliJ의 회사인 JetBrains의 훌륭한 언어이며 2017년 구글이 발표한 안드로이드 개발의 공식언어.
    • 생산성을 향상시킬 뿐만아니라 나중에 안드로이드 개발에 효과적.
    • 요즘 코틀린 + 스프링을 많이 사용한다고 함.
  • Learn Microservices
    • 아키텍쳐는 꾸준히 변화함. 이에 따라 많은 기업들이 모놀리식 개발에서 마이크로 서비스로 변화하고 있음. 최신 흐름을 유지하기 위해 마이크로 서비스 아키텍쳐와 Java로 마이크로서비스를 개발하는 방법을 배워놔야 함.
