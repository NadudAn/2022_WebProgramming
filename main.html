관심 기술 상세 내용

먼저 웹 개발자와 백엔드의 정의에 대하여 알아보려 한다.

  • 웹 개발자
    • HTTP 프로토콜을 커뮤니케이션 매체로 사용하는 웹 페이지, 웹 사이트 등 WWW기반 소프트웨어 개발자 또는 엔지니어.
    • 웹 디자인, 정보설계, 사용자 인터페이스 설계, 프로젝트 관리, 웹 서버 및 데이터베이스 관리, 웹 페이지 코딩 및 프로그래밍 관련 기술을 지님.
    • 전문성을 기준으로 프론트엔드와 백엔드로 분류.

  • 백엔드(Back-end)
    • 사용자에게 보이지 않는 비즈니스 로직 프로그래밍을 하는 개발자.
    • UI나 GUI로 구성된 화면의 통신/요청에 대해 DB 또는 Interface 등을 통해 시스템 구성 실체에 접근.
    • 서버, 응용 프로그램, 데이터베이스를 작동시키는 업무 수행
    • 파이썬, Node.js, PHP7, Ruby, C#, Java, Go 등의 언어 주로 사용.

    • 세부 직무
      • 서버 관리자: 서버 환경설정, 서버구성 관리
      • DB 관리자: DB 설계, Data 관리 및 최적화
      • Back-end Developer: 서버 자체를 코딩
백엔드 개발자가 알아야할 것들
  • API Development
    • Rest API는 오랫동안 API의 표준처럼 사용되어 왔고 서버 개발의 전반적인 흐름을 이해하는데 도움 됨.
  • Languange & Web Framework
    • NodeJS (JavaScript, TypeScript), Ruby On Rails, Python Django, Spring Boot(Java, Kotlin), golang 등
  • Server
    • AWS의 EC2 Instance에 올라가는 이미지들: 주로 Linux 기반의 OS.
    • AWS Elastic Beanstalk: 기본 인프라 구성, 배포, 애플리케이션 버전 제어, 자동 확장 기능까지 전체 시스템을 구축.
  • Network
    • 서버 동작 전체를 잘 이해 하기 위해서 네트워크에 대한 기본 지식 필요.
    • HTTP, DNS, EC2에 접속하기 위한 SSH 연결, 서버가 올라가기 위한 기본 네트워크 구성 (VPC, Routing Table, Subnet)등
  • DB
  • Distributed Storage
    • 서비스에 필요한 서비스를 저장하는 공간.
